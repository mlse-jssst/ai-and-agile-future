# 新バージョンのリリース準備

新しいバージョン `$ARGUMENTS` のリリース準備を行う。

## 手順

1. 引数からバージョン番号と発行年月日を取得する（例: `v3 2026-07-31`）
2. 前バージョンの番号を特定する（Gitタグ一覧から最新のものを取得）

### ファイルの作成・更新

3. `research/$ARGUMENTS のバージョン番号/` フォルダを作成し `.gitkeep` を配置
4. `changelog/` に前バージョンからの変更概要ファイルを作成（テンプレートに従う）:

```markdown
# v{前} → v{新} 変更概要（{発行年月}）

## 期間中の主な動向
- （記入してください）

## 主な変更点
- （記入してください）

## 変更の背景・考察
- （記入してください）

## 次版に向けた課題
- （記入してください）
```

5. `report/main.md` のフロントマターを更新（version と date）
6. `README.md` を更新:
   - 冒頭の「最新版」表記を新バージョン・発行日に変更
   - 更新履歴テーブルに新しい行を追加（「主な変更」欄は空欄のままでよい）

### 確認

7. 変更内容の一覧を表示して完了を報告する

## 使い方の例

```
v3 2026-07-31
```
