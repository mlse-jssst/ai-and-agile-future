# 新バージョンのリリース準備

## 入力

ユーザーから **バージョン番号** と **発行日** を取得する（例: `v3 2026-07-31`）。スペース区切りの2トークンを想定する。

## 手順

1. **前バージョンの特定**  
   `git tag` などで既存タグを確認し、新バージョンの直前の版番号を決める。

2. **`research/{新バージョン}/` の作成**  
   例: `research/v3/`。フォルダを作成し `.gitkeep` を置く。

3. **`changelog/` に変更概要ファイルを追加**  
   ファイル名は `changelog/v{前}-to-v{新}.md`（例: `v2-to-v3.md`）。本文は次のテンプレートに従う。

```markdown
# v{前} → v{新} 変更概要（{発行年月}）

## 期間中の主な動向
- （記入してください）

## 主な変更点
- （記入してください）

## 変更の背景・考察
- （記入してください）

## 次版に向けた課題
- （記入してください）
```

4. **`report/main.md`**  
   YAML フロントマターの `version` と `date` を新バージョン・発行日に合わせて更新する。

5. **`README.md`**  
   - 冒頭の「最新版」表記を新バージョン・発行日に合わせる。  
   - 更新履歴テーブルに行を追加する（「主な変更」欄は空欄のままでよい）。

6. **完了報告**  
   変更したファイルの一覧を示して完了とする。

## 例

```
v3 2026-07-31
```
