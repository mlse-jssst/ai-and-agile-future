# 調査資料の追加

## 入力

ユーザーから **資料の種類** と **タイトル** を取得する（例: `interview 田中氏ヒアリング`）。先頭トークンが種類、以降がタイトルとみなす。

## 手順

1. **次バージョンの推定**  
   最新の Git タグを確認する（例: `v2` が最新なら、作業中は `v3` 向けとして `research/v3/` を使う）。

2. **`research/v{次バージョン}/` にファイルを作成する**

### 種類が `interview` の場合

- **ファイル名**: `interview-{日付}-{短い識別子}.md`（日付は実施日または作成日、識別子はタイトルから短く付ける）

```markdown
# インタビュー: {タイトル}（{今日の日付}）

- 参照レポート: [v{現行タグ}](https://github.com/mlse-jssst/ai-and-agile-future/blob/v{現行タグ}/report/main.md)
- インタビュイー:
- 実施日:
- 実施方法:

## 概要


## 詳細メモ


## レポートへの反映事項

```

### 上記以外の種類

- **ファイル名**: `{種類}-{日付}-{短い識別子}.md`  
- 中身は空の Markdown から始めてよい。

3. **完了報告**  
   作成したファイルのパスを示して完了とする。

## 例

```
interview 田中氏ヒアリング
```
